# 期首仕訳

このマニュアルは、現状の期首仕訳の登録方法と制約をまとめたものです。

## できること

- 会計年度の開始日に対して期首仕訳を 1 件登録できる
- 現金、預金、車両運搬具、棚卸資産の開始残高をまとめて登録できる
- 必要な補助科目は自動作成される

## 現在の登録入口

現状の画面上の入口は初期セットアップです。

- `SetupWizard` で現金、銀行口座、その他資産を入力する
- 入力内容は `opening_entries` に変換される
- 会計年度作成後に期首仕訳として登録される

内部的には `OpeningEntryRegistrar` が使われます。

## 登録される仕訳の形

- 取引日: 会計年度の開始日
- 摘要: `期首残高設定`
- 取引フラグ: `is_opening_entry = true`
- 借方: 入力した資産科目
- 貸方: `元入金`

複数の借方明細を入力した場合でも、作成される取引は 1 伝票です。

## 入力ルール

- `sub_account_name` は必須
- `amount` は 0 より大きい整数であること
- 借方に使える勘定科目は次に限る
  - `現金`
  - `定期預金`
  - `その他の預金`
  - `車両運搬具`
  - `棚卸資産`

## 制約

- 同じ会計年度に期首仕訳は 1 回しか登録できない
- すでに期首仕訳がある会計年度には、追加で期首仕訳を登録できない
- 貸方は常に `元入金`

## code から使う場合

```php
$transaction = $fiscalYear->registerOpeningEntry([
    [
        'account_name' => '現金',
        'sub_account_name' => 'レジ現金',
        'amount' => 100000,
    ],
]);
```

## 補足

将来的に、期首仕訳の差替やセットアップ後の専用 UI を追加する場合も、`FiscalYear::registerOpeningEntry()` を利用側の入口として維持し、その内部で専用の登録処理を使う方針です。
